- `convex/executionLogs.ts`: list/append
- `convex/executeWorkflow.ts`: Phase 1 stub action that:
  - creates an execution (`status: running`)
  - writes `__start__` and `__end__` logs
  - completes execution as `success` (or `failed` with error)
- `convex/users.ts`: added later to support bootstrapping a user row
  - `users.bootstrap` mutation: ensures user row exists
  - `users.me` query: debug helper

### 5.3 Auth / tenancy scaffolding
- `convex/lib/auth.ts`:
  - Refactored to handle the difference between query (read-only) and mutation (read/write) DB contexts.
  - Adds optional ‚Äúdev anonymous user mode‚Äù:
    - If Convex env `AGENTROMATIC_DEV_ANON_USER=true`, unauthenticated calls use externalId `"dev_anonymous"`.
    - Important: the first creation of that user row must happen in a mutation (`users.bootstrap` does that).
- Team auth is still MVP-simplified: team access is allowed only when `teams.ownerUserId === currentUserId`.

### 5.4 Removed problematic helper
- `convex/lib/ownership.ts` was deleted because:
  - It imported types that don‚Äôt exist in the current Convex server typings (`DatabaseReader` export mismatch).
  - It attempted generic constraints that don‚Äôt match Convex‚Äôs `Doc/TableNames` constraints.
  - It was not referenced by current modules (workflows/executions/logs already perform inline ownership checks).
